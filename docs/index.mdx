# Welcome to Routemaster!

<Image src="https://openclipart.org/download/286938/Double-Decker-Bus.svg" width="150" alt="" /> 

Routemaster is a Flutter package that makes it easy to implement path-based routing. It takes a path, such as:

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;`/settings/wifi`
     
...and transforms it into a stack of pages, such as:

&nbsp;&nbsp;&nbsp;&nbsp;`[HomePage(), SettingsPage(), WifiSettingsPage()]`

It's ideal for supporting URLs on the web, and easily supporting deep linking.

Head to [getting started](/getting-started) to start your journey!

<Image src="https://docs.flutter.dev/assets/images/docs/development/packages-and-plugins/FlutterFavoriteLogo.png" alt="Flutter Favorite" align="right" width="100" />

## Features

* Simple [declarative navigation](/navigation) with an easy-to-use API
* Incredibly easy support for [tabs and nested navigation](/tabs)
* Multiple route maps: for example one for a logged in user, another for logged out
* [Observers](/observers) to easily listen to route changes
* Covered by over 160 unit, widget and integration tests

Here's the *entire* routing setup needed for an app featuring tabs and pushed routes:


```dart
final routes = RouteMap(
  routes: {
    '/': (_) => CupertinoTabPage(
          child: HomePage(),
          paths: ['/feed', '/settings'],
        ),

    '/feed': (_) => MaterialPage(child: FeedPage()),
    '/settings': (_) => MaterialPage(child: SettingsPage()),
    '/feed/profile/:id': (routeData) => MaterialPage(
      child: ProfilePage(id: routeData.pathParameters['id'])
    ),
  }
);

void main() {
  runApp(
      MaterialApp.router(
        routerDelegate: RoutemasterDelegate(routesBuilder: (context) => routes),
        routeInformationParser: RoutemasterParser(),
      ),
  );
}
```
